# Deep Agents (딥 에이전트)

> 키워드: 딥 에이전트, deepagents, 플래닝, TODO, 서브에이전트, 컨텍스트 격리, 파일시스템, create_deep_agent, 위임

## 얕은 루프의 한계

"LLM + 도구 호출 루프"만으로 구성된 에이전트는 짧은 작업에는 충분하지만,
수십 스텝짜리 리서치·작성 작업에서는 컨텍스트가 도구 출력으로 가득 차
초반의 계획을 잊어버리고 표류한다. 이를 얕은(shallow) 에이전트라 부른다.

## 딥 에이전트의 4요소

1. **플래닝 도구**: 작업 시작 시 TODO 목록을 작성하고 진행하며 갱신한다.
   계획을 컨텍스트에 반복 노출시켜 장기 작업에서도 목표를 잃지 않게 한다.
2. **서브에이전트**: 하위 작업을 별도 에이전트에 위임한다. 서브에이전트는
   자기만의 컨텍스트 창에서 일하고 결과 요약만 부모에게 돌려준다(컨텍스트 격리).
3. **파일시스템**: 중간 산출물을 컨텍스트가 아니라 파일에 저장한다.
   긴 검색 결과나 초안을 파일로 빼면 컨텍스트 창이 오염되지 않는다.
4. **상세한 시스템 프롬프트**: 작업 절차·품질 기준을 길고 구체적으로 명시한다.

## deepagents 라이브러리

`deepagents` 패키지의 `create_deep_agent(tools, system_prompt, ...)`는
위 4요소가 내장된 LangGraph 에이전트를 만들어준다.
`subagents` 인자로 하위 에이전트 목록을 등록하면 부모가 `task` 도구로 위임한다.
서브에이전트는 프롬프트로 정의할 수도 있고, **이미 만들어 둔 LangGraph 그래프**를
그대로 등록할 수도 있어 기존 자산(예: Adaptive RAG 그래프)을 재사용하기 좋다.

## 관측성과의 결합

딥 에이전트 실행은 "플래닝 → 위임 → 종합"의 깊은 트리 구조라서
트레이싱 없이는 디버깅이 사실상 불가능하다. CallbackHandler를 주입하면
부모-서브에이전트 호출이 하나의 트레이스 트리로 묶여 보인다.
