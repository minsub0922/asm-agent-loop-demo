# LangGraph 핵심 개념

> 키워드: State, 상태, 노드, 엣지, 조건부 엣지, StateGraph, 체크포인터, START, END, 그래프

## StateGraph와 상태(State)

LangGraph의 중심은 `StateGraph`다. 그래프 전체가 공유하는 **상태(State)**를
TypedDict로 선언하고, 각 노드는 상태를 입력받아 "바뀐 부분만" 딕셔너리로 반환한다.
상태는 노드 사이를 흐르는 유일한 통신 수단이며, 그래프의 실행 기록 그 자체다.

## 노드(Node)와 엣지(Edge)

- **노드**: 상태를 받아 상태의 일부를 갱신하는 순수 함수. `graph.add_node("이름", 함수)`로 등록한다.
- **엣지**: 노드 간 실행 순서. `graph.add_edge("a", "b")`는 a 다음에 무조건 b를 실행한다.
- **START / END**: 그래프의 진입점과 종료점을 나타내는 예약 노드.

## 조건부 엣지 (Conditional Edge)

조건부 엣지는 **상태를 보고 다음 노드를 런타임에 결정**하는 분기 장치다.
`graph.add_conditional_edges("노드", 판정함수, {"라벨": "다음노드", ...})` 형태로 등록하며,
판정 함수가 반환한 라벨에 따라 실행 경로가 갈라진다.
조건부 엣지로 "이전 노드로 되돌아가는" 경로를 만들면 그래프에 루프가 생긴다.
워크플로가 에이전트로 진화하는 지점이 바로 이 조건부 엣지다.

## 컴파일과 실행

`graph.compile()`은 선언된 노드·엣지를 실행 가능한 앱으로 만든다.
`app.invoke(초기상태)`는 최종 상태를 반환하고, `app.stream(...)`은 노드 실행을
단계별로 스트리밍한다. 라이브 데모나 디버깅에는 stream이 훨씬 유용하다.

## 체크포인터 (Checkpointer)

체크포인터는 매 스텝의 상태 스냅숏을 저장하는 장치다. `compile(checkpointer=...)`로 연결하며,
thread_id 단위로 실행 이력이 쌓인다. 중단·재개(Human-in-the-Loop), 시간 여행 디버깅,
장애 복구가 모두 체크포인터 위에서 동작한다. 개발 단계에서는 인메모리 체크포인터로 충분하다.
